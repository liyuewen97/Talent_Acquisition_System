# 人才管理系统 V1 PRD

## 1. 项目背景

本系统面向创业阶段猎头团队，目标是把分散在本地文件夹、微信、Boss 等渠道中的候选人资料沉淀为可搜索、可协作、可追踪的统一资源库，并支撑团队录入、跟进、管理和后续 AI 能力扩展。

V1 聚焦以下核心价值：

1. 建立统一候选人库，解决简历分散、查找困难的问题。
2. 建立团队协作和跟进记录体系，解决过程不可视的问题。
3. 建立客户和岗位的基础管理能力，解决优先级和任务分配依赖个人记忆的问题。
4. 为后续简历匹配、推荐报告等 Agent 能力预留结构化数据基础。

## 2. 用户角色

### 2.1 管理员

- 查看和编辑全部候选人、客户、岗位数据。
- 查看全部跟进记录和全部候选人编辑日志。
- 查看管理看板和团队执行情况。
- 管理用户、角色、标签。

### 2.2 猎头助理

- 查看全部候选人。
- 编辑全部候选人。
- 上传简历并发起解析。
- 新增候选人跟进记录。
- 查看候选人相关操作日志和动态。
- 维护客户、岗位的简版信息。

### 2.3 实习生

- 仅查看自己跟进或自己录入的候选人。
- 新增自己的跟进记录。
- 上传简历并补充基础信息。
- 不可查看全量候选人库。

## 3. 核心业务流程

### 3.1 候选人入库流程

1. 团队成员在 Boss、微信等渠道获取候选人简历。
2. 上传 PDF 简历。
3. 系统提取文本并调用简历解析 Agent。
4. 系统生成结构化候选人档案草稿。
5. 用户确认或修正解析结果后入库。
6. 为候选人设置标签、当前跟进人和来源。

### 3.2 候选人跟进流程

1. 用户打开候选人详情页。
2. 新增跟进记录，填写跟进人、方式、内容、下次跟进时间。
3. 系统记录动态并更新最近跟进时间。
4. 管理员在看板和动态页查看团队跟进情况。

### 3.3 客户与岗位流程

1. 管理员或助理录入客户信息。
2. 在客户下新增岗位。
3. 维护岗位优先级、状态和要求。
4. 管理员在首页查看本周重点客户和岗位。

## 4. V1 目标与边界

### 4.1 V1 必做目标

- 候选人统一管理。
- 简历上传与解析入库。
- 标签和基础搜索。
- 候选人跟进记录。
- 候选人编辑日志可视化。
- 客户和岗位简版管理。
- 管理员首页看板。

### 4.2 V1 暂不做

- 智能推荐报告。
- 完整人岗自动匹配闭环。
- 面试流程复杂编排。
- 财务、回款、佣金结算。
- 手机端适配。
- 复杂审批流。

## 5. 功能需求

### 5.1 登录与权限

#### 目标

实现基础身份认证和角色权限控制。

#### 功能点

- 用户登录、退出登录。
- 基于角色控制菜单、列表、详情页访问权限。
- 实习生仅能查看自己录入或自己跟进的候选人。
- 管理员和猎头助理可查看全部候选人。

### 5.2 候选人管理

#### 目标

将候选人信息统一沉淀为结构化档案，支持全局检索和团队协作。

#### 功能点

- 候选人列表页。
- 候选人详情页。
- 新建、编辑候选人。
- 候选人头像或基础识别信息展示。
- 当前跟进人字段。
- 候选人状态字段。
- 来源字段，如 Boss、微信、转介绍、主动投递。
- 最近跟进时间展示。

#### 列表页字段建议

- 姓名
- 手机号
- 邮箱
- 性别
- 年龄
- 当前公司
- 当前岗位
- 毕业年份
- 学历
- 城市
- 当前跟进人
- 标签
- 最近跟进时间
- 来源
- 创建时间

#### 详情页模块建议

- 基础信息
- 教育经历
- 工作经历
- 项目经历
- 标签
- 简历文件
- 跟进记录
- 编辑日志

### 5.3 教育经历

#### 目标

支持候选人多段教育经历，避免只保留单一学校和学历信息。

#### 功能点

- 一位候选人支持多段教育经历。
- 每段教育经历可新增、编辑、删除。
- 支持入学时间、毕业时间。
- 支持学校、学历、专业。
- 在候选人详情页完整展示。

#### 每段教育经历字段

- 学校
- 学历
- 专业
- 入学时间
- 毕业时间
- 是否最高学历

### 5.4 标签管理

#### 目标

支持业务团队快速定义候选人标签体系，提高检索效率。

#### 功能点

- 标签新增、编辑、停用。
- 候选人绑定多个标签。
- 支持自定义标签。
- 候选人列表按标签筛选。

### 5.5 搜索能力

#### 目标

让团队快速从资源库中找到可跟进候选人。

#### 功能点

- 关键字搜索。
- 按姓名、手机号、微信、邮箱搜索。
- 按当前公司、当前岗位搜索。
- 按标签搜索。
- 按当前跟进人搜索。
- 按来源、城市、状态筛选。

### 5.6 简历上传与解析 Agent

#### 目标

将 PDF 简历自动转成结构化候选人档案，降低录入成本。

#### V1 策略

采用“文件提取 + LLM 结构化抽取 + 人工确认”的方案，不追求一次性全自动入库。

#### 功能点

- 上传 PDF 简历。
- 保存原始文件。
- 提取简历文本。
- 调用解析服务输出结构化结果。
- 用户校对后确认入库。
- 保存原始解析结果，便于后续优化。

#### V1 必抽字段

- 姓名
- 手机号
- 微信
- 邮箱
- 出生日期
- 当前公司
- 当前岗位
- 城市
- 总工作年限
- 教育经历
- 最近几段工作经历
- 项目经历
- 技能关键词或标签建议

### 5.7 跟进记录

#### 目标

形成标准化、可追踪的候选人跟进体系。

#### 功能点

- 在候选人详情页新增跟进记录。
- 记录跟进人、跟进方式、跟进内容、下次跟进时间。
- 按时间倒序展示跟进历史。
- 提供最近更新动态视图。

#### 跟进方式建议

- 电话沟通
- 微信沟通
- 视频会议
- 现场见面

### 5.8 候选人编辑日志

#### 目标

保留所有编辑行为，支撑协作可追溯和管理可视化。

#### 功能点

- 候选人每次编辑都写入日志。
- 记录编辑人、时间、字段、修改前、修改后。
- 候选人详情页可查看日志列表。
- 支持按时间筛选。

#### 日志场景

- 编辑基础信息
- 修改标签
- 修改当前跟进人
- 更新教育经历
- 上传或替换简历

### 5.9 客户管理

#### 目标

建立简版客户库，为岗位管理和管理者优先级判断提供基础信息。

#### 功能点

- 客户列表页。
- 新建、编辑客户。
- 基础字段维护。
- 显示客户下岗位数量。

#### 客户字段建议

- 公司名称
- 行业
- 城市
- 合作状态
- 优先级
- 负责人
- 备注

### 5.10 岗位管理

#### 目标

以简版方式沉淀客户招聘需求，支撑管理者查看重点岗位。

#### 功能点

- 在客户下新增岗位。
- 编辑岗位基础要求。
- 维护优先级和状态。
- 在首页显示高优先级岗位。

#### 岗位字段建议

- 所属客户
- 岗位名称
- 城市
- 薪资范围
- 需求描述
- 任职要求
- 优先级
- 状态
- 截止时间
- 负责人

### 5.11 管理员首页看板

#### 目标

帮助管理员快速识别本周重点工作和团队执行情况。

#### 功能点

- 今日新增候选人数。
- 本周新增候选人数。
- 最近跟进动态。
- 超过 N 天未跟进候选人。
- 高优先级岗位列表。
- 各成员录入和跟进统计。

## 7. 非功能要求

- 电脑端优先。
- 默认支持团队协作。
- 关键编辑操作必须留痕。
- 文件上传后需保存原文件。
- 所有时间字段保留创建时间和更新时间。
- 结构化解析结果需保留原始响应，便于优化 Agent。

## 8. 技术建议

- 前端：Next.js
- 后端：Next.js Route Handlers 或 Server Actions
- 数据库：PostgreSQL
- ORM：Prisma
- 文件存储：本地开发先使用本地磁盘，后续可切换到对象存储
- 简历解析：PDF 文本提取 + OCR 兜底 + LLM 结构化抽取
